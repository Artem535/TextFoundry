== Обзор библиотеки

TextFoundry — это C++ библиотека для конструирования текстов и промптов из переиспользуемых смысловых блоков. Библиотека предоставляет детерминированный движок для сборки текстов с поддержкой параметризации, версионирования и стилизации.

=== Основные принципы

* **Engine-first**: Движок — самостоятельная библиотека без зависимостей от UI
* **No magic**: Все преобразования детерминированы, воспроизводимы и тестируемы
* **Control over convenience**: Контроль над текстом важнее автоматизации

=== Пространство имён

Все классы и функции находятся в пространстве имён `tf`.

[source,cpp]
----
#include <textfoundry/engine.h>

using namespace tf;
----

== Основные типы и псевдонимы

=== Идентификаторы

[cols="1,2,3", options="header"]
|===
| Тип | Определение | Описание

| `BlockId` | `std::string` | Глобально уникальный идентификатор блока (нотация `domain.subdomain.name`)
| `CompositionId` | `std::string` | Уникальный идентификатор композиции
| `ParamValue` | `std::string` | Значение параметра
| `Params` | `std::unordered_map<std::string_view, ParamValue>` | Карта параметров
|===

=== Версионирование

[source,cpp]
----
struct Version {
    uint16_t major = 0;  // Breaking changes
    uint16_t minor = 0;  // Backward compatible additions

    std::string toString() const;  // "major.minor"
};

// Поддерживает сравнение
Version v1{1, 0};
Version v2{1, 2};
assert(v1 < v2);
----

== Перечисления

=== BlockType

Категория блока (используется как тег/метаданные):

[cols="1,2", options="header"]
|===
| Значение | Назначение

| `Role` | Роли для LLM (system, user, assistant)
| `Constraint` | Ограничения и требования
| `Style` | Стилистические параметры
| `Domain` | Предметная область
| `Meta` | Мета-информация
|===

=== BlockState

Состояние жизненного цикла:

[cols="1,2", options="header"]
|===
| Значение | Описание

| `Draft` | Редактируемая версия
| `Published` | Опубликованная, неизменяемая версия
| `Deprecated` | Устаревшая, но доступная для чтения
|===

=== ErrorCode

Коды ошибок:

[cols="1,2", options="header"]
|===
| Код | Описание

| `MissingParam` | Обязательный параметр не предоставлен
| `InvalidParamType` | Несоответствие типа параметра
| `UnknownParam` | Параметр не определён в схеме
| `VersionRequired` | BlockRef должен указывать версию
| `VersionNotFound` | Запрошенная версия не существует
| `BlockNotFound` | Блок с данным ID не найден
| `CompositionNotFound` | Композиция с данным ID не найдена
| `InvalidStateTransition` | Недопустимый переход состояния
| `TemplateSyntaxError` | Ошибка синтаксиса шаблона
| `StorageError` | Ошибка хранилища
| `Success` | Успех
|===

== Примеры использования

=== Пример 1: Создание и рендеринг блока

[source,cpp]
----
#include <textfoundry/engine.h>
#include <iostream>

using namespace tf;

int main() {
    Engine engine;

    // Создание блока
    auto greeting = BlockDraftBuilder("greeting.formal")
        .withTemplate(Template("Уважаемый {{name}}, добро пожаловать!"))
        .withDefault("name", "клиент")
        .withLanguage("ru")
        .build();

    // Рендеринг с параметром
    RenderContext ctx;
    ctx.withParam("name", "Алексей");

    auto result = engine.renderBlock("greeting.formal", ctx);
    if (result.hasValue()) {
        std::cout << result.value() << std::endl;
        // Вывод: "Уважаемый Алексей, добро пожаловать!"
    }

    return 0;
}
----

=== Пример 2: Создание композиции

[source,cpp]
----
// Создание блоков
auto headerBlock = engine.createBlockDraft("report.header");
headerBlock.setTemplate(Template("# Отчёт за {{period}}"));
headerBlock.setDefaults({{"period", "текущий период"}});
engine.saveBlock(headerBlock);
engine.publishBlock("report.header", Version{1, 0});

auto bodyBlock = engine.createBlockDraft("report.body");
bodyBlock.setTemplate(Template("{{content}}"));
engine.saveBlock(bodyBlock);
engine.publishBlock("report.body", Version{1, 0});

// Создание композиции
auto report = engine.createCompositionDraft("monthly.report");
report.addBlockRef("report.header", Version{1, 0}, {{"period", "Q3 2024"}});
report.addSeparator(SeparatorType::Paragraph);
report.addStaticText("## Основные показатели\n\n");
report.addBlockRef("report.body", Version{1, 0}, {{"content", "Рост на 15%"}});

engine.saveComposition(report);
engine.publishComposition("monthly.report", Version{1, 0});

// Рендеринг
auto result = engine.render("monthly.report");
if (result.hasValue()) {
    std::cout << result->text << std::endl;
}
----

// Блок с дефолтами
auto block = BlockDraftBuilder("notification")
    .withTemplate(Template("{{urgency}}: {{message}}"))
    .withDefault("urgency", "Информация")
    .build();

// Composition с локальным переопределением
auto comp = engine.createCompositionDraft("alert");
comp.addBlockRef("notification", Version{1, 0}, {{"urgency", "Внимание"}});
engine.publishComposition("alert", Version{1, 0});

// Runtime параметр имеет наивысший приоритет
RenderContext ctx;
ctx.withParam("urgency", "КРИТИЧНО");

auto result = engine.render("alert", ctx);
// Результат: "КРИТИЧНО: {{message}}"
// (message не задан → ошибка или значение по умолчанию)
----

=== Иерархия параметров

Параметры разрешаются в порядке приоритета (от низкого к высокому):

1. **Block Defaults** — значения по умолчанию в Block
2. **Composition Local Override** — `localParams` в BlockRef
3. **Runtime Context** — параметры, переданные в `render()`
