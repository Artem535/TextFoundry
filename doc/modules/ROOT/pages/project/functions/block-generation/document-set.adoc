= Block Generation Document Set

== 1. Покрытие набора документов

Для `Block Generation` сейчас покрыты:

* продуктовый intent — `PRD`
* архитектурное решение — `ADR`
* системный срез — `System Design`
* интеграционный контракт — `Service Spec`
* оценка качества — `Evaluation Plan`
* rollout discipline — `Delivery Plan`
* эксплуатация — `Runbook`
* security boundary — `Security Note`
* prompt governance — `Prompt Template`

== 2. Что уже согласовано между документами

В текущем наборе согласованы:

* explicit authoring nature функции;
* separation between generation and publish;
* `IBlockGenerator` as adapter boundary;
* review-before-save workflow;
* distinction between create mode and revise mode.

== 3. Оставшиеся пробелы

Что ещё можно усиливать в будущем:

* формальный benchmark set;
* versioned prompt governance;
* curated revise corpus;
* явные примеры representative failures.

== 4. Следующие обновления

Следующими полезно поддерживать в актуальном состоянии:

* evaluation corpus;
* prompt policy references;
* links between ADR, prompt template and real provider implementation.
