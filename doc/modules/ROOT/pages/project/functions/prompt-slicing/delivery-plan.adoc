= Prompt Slicing Delivery Plan

== 1. Область поставки

В поставку входят:

* long-prompt decomposition workflow;
* update mode с preservation hints;
* deterministic tagged-section fast path;
* review and publish-all workflow in GUI;
* batch validation semantics.

== 2. Текущий статус

Текущее состояние:

* основной workflow уже реализован в GUI;
* create mode и update mode уже существуют;
* нужен более формальный acceptance corpus и quality discipline.

== 3. Рабочие потоки и владельцы

Основные workstreams:

* AI owner
  Slicing prompt policy, preservation behavior, update heuristics.

* engine owner
  Batch generation contract and validation.

* GUI owner
  Validation, preview UX, publish-all workflow.

== 4. Карта зависимостей

Ключевые зависимости:

* source prompt quality;
* generator adapter;
* `BlockSliceViewModel`;
* reusable ids/summaries for update mode;
* representative legacy prompt set.

== 5. Этапы rollout

Следующие шаги:

1. Собрать набор реальных prompt examples.
2. Разделить acceptance corpus на create mode, update mode и tagged fast path.
3. Зафиксировать quality gates для preservation behavior.
4. Уточнить collision-handling expectations.
5. Проверить publish-all behavior на representative cases.

== 6. Блокеры и риски

Основные блокеры:

* слабая representative dataset coverage;
* зависимость от source prompt quality;
* пока нет formal benchmark set для update-mode preservation.

Риски rollout:

* модель будет слишком агрессивно перестраивать структуру;
* invalid ids будут снижать полезность результата;
* review cost окажется слишком близок к ручной миграции.

== 7. Критерии готовности

Функция считается достаточно готовой, если:

* curated prompt set prepared;
* duplicate-id validation tested;
* update-mode preservation criteria agreed;
* tagged inputs часто обрабатываются без AI regressions;
* publish-all workflow стабилен и понятен пользователю.

== 8. Следующий шаг после текущей итерации

Практически следующая полезная работа:

* связать evaluation corpus с реальными legacy prompts;
* отдельно отслеживать rejected update-mode outputs;
* усилить quality review по structure preservation.
