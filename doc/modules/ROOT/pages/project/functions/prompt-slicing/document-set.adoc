= Prompt Slicing Document Set

== 1. Покрытие набора документов

Для `Prompt Slicing` сейчас покрыты:

* `PRD`
* `ADR`
* `System Design`
* `Service Spec`
* `Evaluation Plan`
* `Delivery Plan`
* `Runbook`
* `Security Note`
* `Prompt Template`

== 2. Что уже согласовано между документами

Согласованы:

* batch nature функции;
* distinction between create mode and update mode;
* deterministic tagged-section fast path;
* review-before-publish workflow;
* preservation-aware update semantics.

== 3. Оставшиеся пробелы

Что в будущем можно усиливать:

* benchmark examples для slicing quality;
* negative cases catalog;
* более явные примеры publish semantics для update-mode reuse;
* curated rejected-batch corpus.

== 4. Следующие обновления

Полезные следующие обновления:

* связать representative prompts с evaluation corpus;
* поддерживать в синхроне create/update/fast-path quality rules;
* при необходимости выделить отдельный update-mode appendix.
