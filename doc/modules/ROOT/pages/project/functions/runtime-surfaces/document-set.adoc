= Runtime Surfaces Document Set

== 1. Сводка покрытия

Набор документов покрывает:

* продуктовую роль `C++ / Python local / remote` surfaces;
* distinction между local и remote boundary;
* parity policy;
* relation к `Prompt Server`;
* rollout order;
* security и release implications.

== 2. Что уже зафиксировано хорошо

Сильные стороны текущего набора:

* capability теперь описана не только как архитектурный `ADR`;
* честно разделены `реализовано сейчас` и `planned`;
* `Python` закреплён как first-class surface, но не как новый source of truth;
* `Prompt Server` увязан с multi-language story без смешения ролей.

== 3. Сквозные пробелы

Пока остаются открытыми:

* конкретная binding technology для `Python local`;
* packaging strategy для local SDK;
* transport/serialization strategy для remote clients;
* точная compatibility policy по версиям API и SDK;
* реализация automated parity suite.

== 4. Следующие обновления

* `System Design`
