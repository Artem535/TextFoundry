= Runtime Surfaces Prompt Template

== 1. Статус

У этой функции нет собственного production prompt.

Причина:

* `Runtime Surfaces` описывает language/runtime boundary и public contracts;
* capability не вводит отдельный LLM prompt layer;
* local и remote clients используют deterministic engine/server semantics, а не
  самостоятельный prompt orchestration.

== 2. Что здесь считается входом вместо prompt

Для этой функции важны не prompts, а контракты:

* local method families;
* remote request/response contracts;
* error semantics;
* compatibility rules.

== 4. Следствие для реализации

Если позже появятся:

* codegen helpers;
* AI-assisted API documentation;
* LLM-driven migration assistants;

то для них нужно будет создавать отдельный prompt-oriented document, а не
расширять этот файл задним числом.
