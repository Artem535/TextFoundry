= Block Authoring Runbook

== 1. Операционный обзор

* Владелец функции: владелец библиотеки блоков и интерфейса редактирования
* Критичность: высокая для повседневной работы с библиотекой блоков
* Модель поддержки: инженерная поддержка в рабочее время и во время релизных
  проверок

== 2. Владение и поддержка

* Основной владелец: владелец сценария авторинга блоков
* Вторичный владелец: владелец движка и версионирования
* Дополнительный владелец: владелец AI-генерации блоков

== 3. Ключевые сигналы

* ошибки открытия блоков
* ошибки публикации новой версии
* ошибки удаления
* сообщения о потере несохранённых данных
* частые отказы AI-режимов

== 4. Частые сбои

* выбранный блок не найден
* неверный формат значений по умолчанию
* попытка удалить блок, используемый в композиции
* отсутствие AI-настроек
* AI вернул непригодный черновик

== 5. Реакция на инцидент

* проверить, что блок существует и версия корректна
* проверить, что проблема воспроизводится без AI
* если проблема в удалении, проверить ссылки из композиций
* если проблема в публикации, убедиться, что черновик не потерян
* если проблема только в AI, перевести пользователя на ручное редактирование

== 6. Режимы деградации

* без AI функция должна оставаться полностью работоспособной в ручном режиме
* при ошибке удаления библиотека остаётся без изменений
* при ошибке публикации пользователь остаётся в форме с введёнными данными

== 7. Восстановление и проверка

* перезагрузить дерево блоков
* открыть известный блок
* создать тестовую версию
* проверить список версий
* убедиться, что AI-режим отключается без поломки ручного сценария

== 8. Эскалация

* руководитель: владелец сценария авторинга блоков
* владелец ядра: владелец движка и публикации
* владелец AI: владелец генерации и ревизии блоков
