= Composition Authoring Runbook

== 1. Операционный обзор

* Владелец функции: владелец редактора композиций и ядра версионирования
* Критичность: высокая для повседневной сборки и сопровождения промптов
* Модель поддержки: инженерная поддержка во время активной разработки и релизов

== 2. Владение и поддержка

* Основной владелец: владелец сценария авторинга композиций
* Вторичный владелец: владелец ядра и репозитория композиций
* Дополнительный владелец: владелец библиотеки блоков

== 3. Ключевые сигналы

* ошибки открытия композиции
* ошибки публикации новой версии
* ошибки загрузки списка версий
* ошибки ссылок на блоки
* жалобы на потерю порядка фрагментов

== 4. Частые сбои

* не выбрана композиция
* некорректная версия блока
* ошибка загрузки опубликованной версии
* ошибка публикации новой версии
* потеря ожидаемого состояния после операций над фрагментами

== 5. Реакция на инцидент

* проверить наличие выбранной композиции и версии
* проверить, воспроизводится ли проблема на известной композиции
* если проблема в ссылках, проверить библиотеку блоков и версии блоков
* если проблема в редакторе, проверить операции вставки, вырезания и
  перестановки по шагам

== 6. Режимы деградации

* если дополнительные функции рядом с редактором временно недоступны, базовый
  редактор композиции должен продолжать работать
* при ошибке публикации пользователь должен остаться в редакторе
* при ошибке удаления список не должен показывать ложный успех

== 7. Восстановление и проверка

* перезагрузить список композиций
* открыть тестовую композицию
* выпустить новую тестовую версию
* проверить, что порядок фрагментов сохранился
* проверить, что версия изменилась ожидаемым образом

== 8. Эскалация

* руководитель: владелец сценария авторинга композиций
* владелец ядра: владелец публикации и версий
* владелец блоков: владелец зависимостей композиции от библиотеки блоков
