= Prompt Server Prompt Template

Статус: placeholder для planned capability.

Для server capability отдельный runtime prompt-template может вообще не
понадобиться. Страница сохранена ради единообразия document pipeline и чтобы
заранее зафиксировать ограничения.

== 1. Назначение возможного prompt layer

Если в будущем capability введёт runtime wrappers или server-side prompt
envelopes, они должны:

* оставаться в deterministic published-asset contract;
* не становиться скрытым authoring layer;
* не подтягивать draft semantics.

== 2. Ограничения

Если server когда-либо будет поддерживать built-in wrappers, они должны быть:

* deterministic;
* versioned;
* published-only;
* contract-stable;
* совместимыми с базовым render contract.

== 3. Чего здесь сознательно нет

Пока в документе сознательно нет:

* runtime prompt schema;
* примеров server-side prompt assembly;
* format-specific instructions.

Причина:

* capability пока не реализована, и выдумывать prompt contract раньше времени
  было бы ложной детализацией.
