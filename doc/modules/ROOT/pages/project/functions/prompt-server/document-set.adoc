= Prompt Server Document Set

Статус: planned capability.

== 1. Покрытие набора документов

Для `Prompt Server` сейчас есть placeholder-документы по тем же разделам, что и
для реализованных функций:

* `PRD`
* `ADR`
* `System Design`
* `Service Spec`
* `Evaluation Plan`
* `Delivery Plan`
* `Runbook`
* `Security Note`
* `Prompt Template`

== 2. Что уже согласовано между документами

Уже согласованы:

* published-only delivery boundary;
* отсутствие authoring surface;
* deterministic base contract;
* planned status без ложных заявлений о runtime implementation.

== 3. Текущие пробелы

Пробелы пока сознательные:

* нет final API shape;
* нет deployment details;
* нет authn/authz specifics;
* нет executable runtime.

== 4. Следующие обновления

При начале реализации нужно:

* convert placeholders to final contracts;
* align service spec with deployment and security decisions;
* обновить runbook и evaluation plan на реальные runtime properties;
* при необходимости выделить отдельные ADR по authn/authz и compatibility.
