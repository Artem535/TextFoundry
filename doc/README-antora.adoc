= Antora Setup

== Local Build

[source,bash]
----
npm install
npm run docs:sync
npm run docs:build
----

The generated site is written to `public/`.

== GitLab Pages

The repository already contains a `pages` job in `.gitlab-ci.yml`.

On the default branch this job:

* installs Node dependencies;
* runs `npm run docs:build`;
* copies the product one-pager to `public/presentation/index.html`;
* publishes the generated `public/` directory to GitLab Pages.

Standalone HTML assets placed under `doc/modules/ROOT/assets/attachments/`
become part of the published site as static files and can be linked from Antora
pages.

For the current product presentation this means:

* attachment form remains available inside the Antora site;
* a dedicated GitLab Pages route is also published at `/presentation/`.

== Structure

* `doc/antora.yml` — Antora component descriptor
* `doc/modules/ROOT/nav.adoc` — site navigation
* `doc/modules/ROOT/pages/*.adoc` — thin wrapper pages
* `doc/modules/ROOT/partials/generated/*.adoc` — generated partial copies used by Antora
* `antora-playbook.yml` — root site playbook
* `scripts/sync_antora_docs.sh` — sync script for source documents

== Source Documents

The following files remain the primary source documents:

* `doc/prd.adoc`
* `doc/status.adoc`
* `doc/spec-kit.adoc`
* `doc/doc.adoc`

Before each Antora build, `scripts/sync_antora_docs.sh` copies these files into
Antora partials so that the site build can include them using supported Antora
resource families.
