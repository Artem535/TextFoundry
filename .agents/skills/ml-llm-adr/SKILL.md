# ML / LLM ADR

Read:

* `../../../ML LLM Delivery Pipeline/03 ADR Template.md`

Use this skill for one decision at a time.

## Cover

* context
* decision drivers
* alternatives considered
* chosen option and rationale
* architecture slice impacted by the decision
* consequences, risks, and mitigations
* linked documents that must be updated

## Guardrails

* Keep the ADR narrow and decision-shaped.
* Do not turn it into a full system design doc.
